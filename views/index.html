User Route üë§
POST /user/register

Registers a user with a username and password

Request Body
username: must be alphanumeric, minimum 3 chars, maximum 30 chars, required
password: required
{ username: String, password: String, }
Response Body
{ data: { token: "JWT_TOKEN" } }
POST /user/login

Logs in a registered user

Request Body
Must match record in database: { username: String, password: String, }
Response Body
{ data: { token: "JWT_TOKEN" } }
Posts Route üìù
IMPORTANT NOTE: None of the Post Routes will work unless you add the auth-token (received from registering/logging in) header to the requests
GET /posts

Returns all post records

Response Body
{ _id: String, title: String, description: String, date: Date }
GET /posts/:postId

Returns a specific post record

Parameters
  • :postId The ID of the post you want to retrieve information for
POST /posts

Submits a post to the database

Request Body
  • { title: String, description: String, author: String, }
PATCH /posts/:postId

Updates a post in the database

Parameters
  • :postId The ID of the post you want to retrieve information for
Request Body
  • { title: String, }
DELETE /posts/:postId

Removes a post in the database

Parameters
  • :postId The ID of the post you want to retrieve information for
